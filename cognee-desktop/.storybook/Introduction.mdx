## What you can do here

1. **Browse components** in the sidebar. Primitives live under **Components/**, composite stateful patterns under **Patterns/**, and the graph visualisation under **Visualizations/**.
2. **Toggle states** via the **Controls** panel. Change props and watch the component respond.
3. **Check accessibility** in the **Accessibility** panel. axe-core runs on every story; a clean panel passes the a11y gate.
4. **Switch theme** via the toolbar. The light/dark toggle exposes how each component renders in both modes.

## Ship gates

Every story is automatically gated by:

- **Accessibility** via `addon-a11y` (failures surface in the Accessibility panel)
- **TypeScript-strict typing** via the `Meta<typeof Component>` `satisfies` pattern
- **Lint and format** via Ultracite/Biome on every commit

Visual regression (snapshot diff) is **planned for v0.2**, deferred from v0.1. Until that lands, manual visual review of changed stories during code review is the working signal.

## v0.1 coverage

13 components are tracked here for the v0.1 ship.

- **Components/**: Avatar, Password Input, Settings Radio, Dataset Row, Empty State
- **Patterns/**: Confirm Modal, Undo Toast, Inspect Panel, Cognify Overlay, Drop-Zone Overlay, Setup-gate, Auth0 Device-Code Shell
- **Visualizations/**: Graph Canvas (3D force-directed knowledge graph)

Coverage rolls in incrementally. When a component lands here, its behaviour is locked.

## Dark mode

v0.1 ships **light only**. The dark toggle in the toolbar is wired up so dark visuals can be spot-checked as they're built, but the production app does not switch themes at v0.1. Dark mode promotes to a ship target at v0.2 alongside the visual regression infrastructure.

## Running locally

Storybook for Cognee Desktop is local-only for v0.1 (no hosted preview). To review:

```bash
cd cognee-desktop
bun install
bun storybook
```

Then open http://localhost:6006.
